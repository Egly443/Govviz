# Govviz

Long-run performance data for UK government departments, presented in context
rather than in isolation. Each department has a hero trend, a set of core
"competence" measures, optional supporting context, and a per-indicator RAG
snapshot (deliberately **not** reduced to a single league-table grade).

Built with **Vite + React + TypeScript**, **TanStack Router** (file-based
routing), **Tailwind CSS v4**, **Recharts**, and **lucide-react**.

## Departments

HMT (Treasury), DHSC (Health & Social Care), DfE (Education), Home Office,
MoJ (Justice), MoD (Defence), DWP (Work & Pensions), DfT (Transport),
MHCLG (Housing, Communities & Local Government), and Defra (Environment,
Food & Rural Affairs).

> **Real data only.** Every chart shows real data fetched from official sources
> (ONS, World Bank, gov.uk, NHS England, the Environment Agency, DfE EES and
> others) in CI and baked into the production build. There is no fabricated or
> illustrative data: a series with no wired source renders an explicit "no
> source yet" placeholder rather than an invented trend. See `sourceUrl` on each
> series and `scripts/build-data.mjs` for the per-series sourcing, and
> `docs/conformance/` for the AI-readiness conformance suite and series profile.

## Getting started

```bash
npm install
npm run dev      # start the dev server (http://localhost:5173)
npm run build    # production build to dist/
npm run preview  # preview the production build
npm run typecheck
```

> A local build without CI-baked data shows placeholders for every series
> (illustrative fallbacks were removed by design). The live site is fully
> populated because CI fetches the real data on deploy.

```

  main.tsx               # app entry, creates the router
  index.css              # Tailwind v4 import + theme tokens (dark)
  routeTree.gen.ts       # generated by the TanStack Router plugin

    DepartmentTabs.tsx   # sticky Overview + department switcher
    TopNav.tsx           # top navigation bar

  conformance/           # AI-readiness conformance suite + series profile
  blog-open-data-for-ai.md
```

## Overview treemap

`/overview` is the landing page: a whole-of-government heatmap. Each
**department block is sized by its spending** (`spendBn`), and within a block
each indicator tile is **weighted Finviz-style** (hero > core > supporting).
Tile **colour** is a red→amber→green RAG score: where a published target/standard
exists, green means at or beyond that benchmark. Indicators with **no external
benchmark** are scored against their own history and shown **desaturated** (and
labelled in the legend) so an own-range score never reads as a verdict. Unsourced
indicators are greyed with no value. Clicking a tile opens its full `TrendPanel`
chart in a modal. A dot marks indicators that have an official target.

## Routing

`/` redirects to `/overview`. Each department is reachable at `/<code>`, e.g.
`/hmt`, `/dhsc`, `/dfe`, `/home-office`, `/moj`, `/mod`, `/dwp`, `/dft`,
`/mhclg`, `/defra`. Unknown codes render a 404 with a link back to DHSC.
